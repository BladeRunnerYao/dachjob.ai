dachjob.ai Documentation

This folder contains the split requirements and implementation plans for the local-first DACH AI job platform demo. Start with the master PRD, then assign individual module documents to coding agents.

Master Documents

00 Master PRD Project overview, goals, scope, architecture, success criteria. Parallel Agent Plan How to split implementation across multiple coding agents. 2-Minute Demo Script Interview walkthrough and architecture talking points.

Module Requirements

01 Local Docker Runtime Docker Compose, services, environment variables, local startup. 02 Backend API FastAPI foundation, tenant context, health checks, module layout. 03 Data Model Postgres schema, pgvector, migrations, tenant isolation. 04 LLM Gateway DeepSeek integration, prompt versioning, LLM run logging. 05 Job Matching JD parsing, deterministic fit score, DACH-specific rules. 06 RAG Resume Generation Evidence chunks, retrieval, tailored CV, PDF export. 07 Frontend Dashboard Next.js pages, product UI, mock-first frontend plan. 08 Tracker and Autofill Application tracker and safe autofill demo scope. 09 Embedding Provider Strategy Local embeddings, cloud API options, pricing notes, and provider switching.

Repository Layout

Code should be implemented under app/ and infra/. Requirements and plans live under docs/.
